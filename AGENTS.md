## Формат коммитов

- Все коммиты должны соответствовать [Conventional Commits](https://www.conventionalcommits.org/).
- Допустимые типы: `feat`, `fix`, `perf`, `docs`, `refactor`, `test`, `build`, `ci`, `chore`.
- Для breaking changes используй `!` после типа/scope или `BREAKING CHANGE:` в footer сообщения.
- Текст коммита пиши на английском языке, чтобы CHANGELOG и release notes были единообразными.
