# Сценарии end-to-end тестирования (Playwright)

Этот документ описывает основные пользовательские сценарии сервиса **Calendar Booking API**, которые должны быть воспроизведены с помощью Playwright.

Сценарии описаны в формате Gherkin: ключевые слова (`Feature`, `Scenario`, `Given`, `When`, `Then`, `And`, `But`) — на английском, шаги и описания — на русском.

## Окружение

Для воспроизведения сценариев фронтенд должен быть доступен на `http://localhost:5173`.

### Вариант 1: реальный бэкенд

```bash
cd typespec && npm install && npm run compile
cd ../backend && make run
cd ../frontend && cp .env.example .env && npm install && npm run dev
```

Бэкенд будет на `http://localhost:8080`.

> Важно: для работы с фронтендом бэкенд должен быть запущен с переменной `CORS_ALLOWED_ORIGINS=http://localhost:5173`.

### Вариант 2: мок API (Prism)

```bash
cd typespec && npm install && npm run compile
cd ../frontend && cp .env.example .env && npm install && npm run dev:mock
```

Фронтенд на `http://localhost:5173`, мок API на `http://localhost:4010`.

## Feature: Публичная запись на встречу

```gherkin
Feature: Публичная запись на встречу
  Как гость
  Я хочу выбрать подходящий тип встречи, дату и время
  Чтобы записаться на встречу

  Scenario: Гость просматривает список типов событий
    Given в системе есть хотя бы один тип события
    When пользователь открывает главную страницу "/"
    Then отображается заголовок "Book a Meeting"
    And видна карточка типа события с названием и длительностью
    And на карточке есть кнопка "Book"

  Scenario: Гость открывает страницу события и видит доступные слоты
    Given существует тип события "30-min call" с длительностью 30 минут
    And у типа события "30-min call" есть доступные слоты
    When пользователь открывает главную страницу "/"
    And нажимает кнопку "Book" у карточки "30-min call"
    Then открывается страница "/event-types/{id}"
    And отображается название типа события
    And отображается список доступных дат

  Scenario: Гость выбирает дату и видит слоты
    Given существует тип события "30-min call"
    And у типа события "30-min call" есть доступные слоты на дату "2026-06-30"
    When пользователь открывает страницу "30-min call"
    And нажимает кнопку с датой "2026-06-30"
    Then отображается список доступных временных слотов на эту дату
    And каждый свободный слот доступен для выбора
    And каждый занятый слот отображается как недоступный

  Scenario: Гость успешно бронирует свободный слот
    Given существует тип события "30-min call"
    And у типа события "30-min call" есть свободный слот
    When пользователь открывает страницу "30-min call"
    And выбирает дату с доступными слотами
    And нажимает на свободный слот
    And в модальном окне вводит имя "Иван"
    And нажимает кнопку "Confirm booking"
    Then появляется уведомление "Booking confirmed!"
    And модальное окно закрывается
    And выбранный слот становится занятым

  Scenario: Гость получает ошибку при попытке забронировать занятый слот (409)
    Given существует тип события "30-min call"
    And у типа события "30-min call" есть свободный слот
    And другой пользователь занял этот слот после открытия формы
    When пользователь открывает страницу "30-min call"
    And выбирает дату с указанным слотом
    And нажимает на свободный слот
    And в модальном окне вводит имя "Иван"
    And нажимает кнопку "Confirm booking"
    Then появляется уведомление "Slot already taken"
    And модальное окно закрывается
    And список слотов обновляется

  Scenario: Клик по занятому слоту не открывает форму бронирования
    Given существует тип события "30-min call"
    And у типа события "30-min call" есть занятый слот
    When пользователь открывает страницу "30-min call"
    And выбирает дату с занятым слотом
    And нажимает на занятый слот
    Then модальное окно бронирования не открывается
    And поле ввода имени не отображается
```

## Feature: Управление типами событий (админка)

```gherkin
Feature: Управление типами событий (админка)
  Как администратор
  Я хочу управлять типами событий
  Чтобы гости могли записываться на встречи

  Scenario: Админ создаёт новый тип события
    Given пользователь открыл страницу "/admin/event-types"
    When нажимает кнопку "+ New event type"
    And вводит название "Consultation"
    And вводит описание "Detailed consultation"
    And вводит длительность "45"
    And нажимает кнопку "Create"
    Then появляется уведомление "Event type created."
    And новый тип события отображается в таблице
    And на главной странице "/" отображается карточка "Consultation"

  Scenario: Админ редактирует тип события
    Given существует тип события "Consultation"
    And пользователь открыл страницу "/admin/event-types"
    When нажимает кнопку "Edit" у типа события "Consultation"
    And изменяет название на "Consultation Updated"
    And нажимает кнопку "Save"
    Then появляется уведомление "Event type updated."
    And в таблице отображается обновлённое название

  Scenario: Админ удаляет тип события
    Given существует тип события "Consultation Updated"
    And пользователь открыл страницу "/admin/event-types"
    When нажимает кнопку "Delete" у типа события "Consultation Updated"
    And в модальном окне подтверждает удаление, нажав "Delete"
    Then появляется уведомление "Event type deleted."
    And тип события исчезает из таблицы
    And на главной странице "/" карточка "Consultation Updated" не отображается
```

## Feature: Управление бронированиями (админка)

```gherkin
Feature: Управление бронированиями (админка)
  Как администратор
  Я хочу просматривать и отменять бронирования
  Чтобы управлять расписанием

  Scenario: Админ просматривает список бронирований
    Given существует хотя бы одно бронирование
    When пользователь открывает страницу "/admin/bookings"
    Then отображается заголовок "Bookings"
    And в таблице видны гость, идентификатор типа события, начало и окончание встречи
    And на каждой строке есть кнопка "Cancel"

  Scenario: Админ отменяет бронирование
    Given существует бронирование для гостя "Иван"
    And пользователь открыл страницу "/admin/bookings"
    When нажимает кнопку "Cancel" у бронирования гостя "Иван"
    And в модальном окне подтверждает отмену, нажав "Cancel booking"
    Then появляется уведомление "Booking has been cancelled."
    And бронирование исчезает из таблицы
    And соответствующий слот снова становится доступным для бронирования
```
